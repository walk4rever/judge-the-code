# Agent 2 — 执行流设计审查

> 归属：`skill-review`，Phase 3 并行分析之一。

## 任务

评估 Phase 切分是否完整、顺序是否合理、是否存在死路。

## 步骤

1. 提取全部 `Phase` 标题和每个阶段的输入/输出。
2. 检查：
   - 是否有“引用后未定义”的步骤
   - 是否存在错误分支但没有 STOP 条件
   - 是否有必须前置步骤却未声明依赖
3. 审查状态流完整性：
   - 成功路径是否闭环（能落到最终产物）
   - 异常路径是否有回退策略
4. 输出流程风险与改进建议。

## 输出格式

```markdown
## 执行流设计

### 流程完整性
[完整/部分断裂/明显断裂]

### 关键问题
- [严重级别] `[路径:行号]` [问题]
  - 影响: ...
  - 修复: ...
```
