# Agent 3 — 安全边界与注入风险审查

> 归属：`skill-review`，Phase 3 并行分析之一。

## 任务

审查 Prompt Injection 风险和危险操作边界。

## 步骤

1. 检查是否明确区分：
   - 系统约束（不可覆盖）
   - 用户输入（不可信）
2. 搜索高风险模式：
   - 直接拼接用户输入到系统指令
   - 未限制的文件系统/命令执行范围
   - 危险命令无告警或无确认机制
3. 标注风险级别：
   - Critical：可直接造成破坏操作
   - High：高概率被绕过系统约束
   - Medium/Low：可控但建议修复

## 输出格式

```markdown
## 安全边界

### 风险发现
- [严重级别] `[路径:行号]` [风险描述]
  - 可利用方式: ...
  - 缓解建议: ...
```
