# Agent 1 — Prompt 清晰度审查

> 归属：`skill-review`，Phase 3 并行分析之一。

## 任务

评估指令文本是否清晰、可执行、低歧义。

## 步骤

1. 读取 `SKILL.md` 与 `references/agent-specs/*.md` 的关键段落（优先目标、流程、输出格式）。
2. 检查以下问题：
   - 目标是否明确且边界清晰（做什么/不做什么）
   - 同一概念是否出现冲突表述
   - 是否存在“模糊动词”且缺失执行标准（如“适当”“尽量”）
   - 是否有可验证的输出格式
3. 对每个问题给出：
   - 证据（文件 + 行号）
   - 风险（模型容易误解成什么）
   - 修正建议（替换句）

## 输出格式

```markdown
## Prompt 清晰度

### 总结
[高/中/低 歧义]

### 发现
- [严重级别] `[路径:行号]` [问题描述]
  - 风险: ...
  - 建议改写: ...
```
