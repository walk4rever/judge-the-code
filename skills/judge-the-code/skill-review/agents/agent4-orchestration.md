# Agent 4 — Agent 编排/容错/兼容性审查

> 归属：`skill-review`，Phase 3 并行分析之一。

## 任务

审查并行编排是否真实独立、失败降级是否可用、是否过度绑定单一模型特性。

## 步骤

1. 检查并行 Agent 的输入依赖：
   - 若 A 的输出被 B 必需消费，则不应声明并行
2. 检查失败处理：
   - 工具缺失时是否有 fallback
   - 子任务失败时是否可部分完成
3. 检查兼容性：
   - 是否硬依赖单一模型专有语法（如特定标签格式）
   - 是否提供降级或中性表述
4. 给出优先级排序建议（先修复最影响可靠性的问题）。

## 输出格式

```markdown
## 编排与容错

### 并行设计
[合理/存在隐式串行依赖]

### 容错与兼容性问题
- [严重级别] `[路径:行号]` [问题]
  - 后果: ...
  - 修复建议: ...
```
